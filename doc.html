Structure du master

Un master se compose d'intervenants, d'unités, de semestres, d'options, de groupes et de spécialités.

Intervenants

Chaque intervenant est identifiable avec son attribut id. Il se compose d'un nom, d'un prénom, d'une adresse email et d'un site web.

Unités

Une unité est identifiable avec son attribut id. Elle se compose d'un nombre de crédits, d'un résumé, d'un intervenant responsable et d'une liste d'intervenants qui interviennent dans cette unité.

Semestres

Un semestre se compose d'un nom, d'un nombre de crédits, d'un numéro ainsi que d'une liste de références qui va représenter la liste des unités qui le composent.

A un semestre on attribue un identifiant (id).

Options

Une option est identifée avec son attribut id. Elle se compose d'un nom, d'un nombre de crédits, d'un numéro et d'une liste d'unités enseignées dans ce bloc d'options.

Groupes

Un groupe représente plusieurs unités enseignées. Chaque groupe se compose d'un nom, d'un nombre de crédits, d'un numéro et d'une liste d'unités enseignées dans ce groupe d'unités.

Specialités

Une spécialité (identifiée par son attribut id) peut être un parcours ou bien définie par un responsable, un nom, une description puis un ou plusieurs parcours.

Parcours

Un parcours est identifié (par id). Il est composé d'un responsable, d'un nom, d'une description, d'une liste de debouchés, d'une liste de lieux et d'une référence vers la composition de ses (deux) semestres.

Structure du site web

Localisation

Le site web se trouve dans le dossier www

Fonctionnement global

Les pages html sont générées automatiquement par le Makefile en faisant make web qui va alors utiliser une feuille XSL pour transformer le xml en plusieurs documents HTML.

Le framework CSS Bootstrap est utilisé pour rendre le site plus portable.

Chaque intervenant, unité, parcours et spécialité est exporté dans un fichier HTML séparé.

XQuery

La requête se trouve dans le fichier xq.txt, saxon est utilisé pour exécuter la requête.

Java

Le code source Java ue.java permet de naviguer dans la structure à la recherche des noms des unités pour ensuite les afficher dans la sortie standard. Le principe est simple : récupérer un document DOM en mémoire, récupérer les unités, les parcourir pour les afficher.
